# Project Instructions

This `docs` folder is the internal reference area for how this portfolio and blog should be managed.

Use these sections as the source of truth:

- `Workflow/BlogOperations.md` for blog and publishing rules
- `Workflow/PortfolioBehavior.md` for how project work should be presented
- `Workflow/PortfolioChecklist.md` for project review before publishing
- `Workflow/ProjectLifecycle.md` for project capture-to-publish flow
- `Workflow/StoryFramework.md` for case study structure
- `Workflow/EvidenceRules.md` for claim and proof standards
- `Tracking/` for content planning and status tracking
- `Templates/` for reusable writing and project templates
- `Reference/` for supporting files that should not live in the site content folders

Working rule:

- Keep private notes, internal guidance, drafts, and reference material in `docs/`
- Keep public website content in `content/`
- Do not place draft-review or conflict files inside `content/`
